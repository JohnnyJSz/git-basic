What is Git?

A free and open source version control system.

What is Version Control System?

The management of changes to documents, computer programs, large web sites and other collections of information.

Terms

  • Directory: Folder

  • Terminal or Command Line: Interface for text commands

  • CLI: Command Line Interface

  • cd: Change Directory

  • Code Editor: Word processor for writing code

  • Repository: Project, or the folder/place where your projects are kept

  • GitHub: A website to host your repositories online.

Some Git Commands

  • clone -- Bring a repository that is hosted somewhere like GitHub into a folder on your local machine.

    git clone git@github.com/user/repo_example.git

  • status -- Shows all the files that were updated, created or deleted but haven't been saved in a commit yet.

    git status

  • add -- Track your files and changes before we can save it to Git, we have to tell it which files we want to track

    git add file_name / git add .

  • commit -- Save your files locally, the commit it's not live in GitHub, to do so we use git push

    git commit -m 'commit title' -m 'commit description'

  • push -- upload Git commits to a remote repository, like GitHub

    git push origin master

    • origin - the location of our Git repository.

    • master - the branch that we want to push to.

  • pull -- Download changes from remote repository to your local machine, the opposite of push

SSH Keys

In order to push the changes under our account we have to proof to GitHub that we are "the owners of our account" (connect our local machine to GitHub)

How? Using the SSH KEYS
We have to start by generating a key locally using the ssh-keygen command

ssh-keygen

the type of encryption

ssh-keygen -t rsa

the strength of encryption

ssh-keygen -t rsa -b 4096

and our GitHub email address

ssh-keygen -t rsa -b 4096 -C "email@example.com"

Now the default file for the ssh key is inside our user directory in the .ssh/ directory and its called id_rsa

Now the key is generated. If we check our key we find out there is two keys (both the same but one with the extension .pub). Is that one the one we are going to upload to our GitHub interface.

We have to print out and copy the public key. It could start with 'ssh-rsa' and it is a very long chain of characters ending with the email linked to our GitHub account

Once we have the key on the clipboard, we have to go to GitHub web site > user > settings > SSH and GPG keys > New SHH key > here we have to paste the key.

The only thing left to do it's to make sure that our local Git CLI knows about the key we just generated. For that simply follow the steps in the following link Generating a new SHH key and adding it to the shh-agent

Git Push

To push to GitHub we have to type in the Terminal git push origin master

origin is an option set for us here and stands for the location of our git repository.

master is the branch that we want to push to.

Start a repository locally

So far we have been doing commits to practice, that was part of a demonstration of how to create a new git repository using GitHub but what about if we start a repository locally?


Lets create a new folder, separated from the previous repository, lets called it demo-repo and lets move into it. Now this is not a Git repository yet, nothing is in there.

Lets add a test file and add some content to it.

To turn that folder and its content into a git repo we have to type git init.

After some changes we follow the same steps to commit status / add / commit / push origin master and we are going to face an error
"fatal: 'origin' does not appear to be a git repository"
and
"fatal: Could not read from remote repository."

This is because we created it locally and there is no repo on GitHub to push to, it is not connected to anything. So we have to create that connection.

The easiest way to do this is by creating an empty git repo up on github.com, once we have it we need to copy the SSH key, after that we hay to type in the Terminal git remote add origin "shh key from github.com" and then we can check the repositories that we have connected to this repo using the command git remote -v

Now we have all set up we can use git push origin master, there is a shortcut for that but we have to set something called an upstream meaning this is where I want to push it to by default, we can set the shortcut with git push -u origin master and then the next time in the future we can use git push only.

After that if we refresh the github website we will find the repository updated.
